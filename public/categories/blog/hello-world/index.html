天际私语：博客的毁灭与重生

这是我第二次搭建博客。

🕰️ 前传：老旧笔记本上的 WordPress

上大学的时候我看见其他学长写的博客，觉得很有趣，于是准备尝试搭一个博客。

我的第一个博客用的是 WordPress 框架。当时我正在做 3D 打印机，手里正好收了一台 2011 年的旧笔记本电脑 做上位机，本着物尽其用的原则，我把第一个博客部署在了那上面。

💥 崩溃连锁：硬件的“花式死亡”

有趣的事情很快就发生了。那台老电脑的处理器、内存和硬盘仿佛约好了一样，接连出现了问题。第一周运行的时候尚且安好，到了第二周，各种诡异的故障开始轮番上阵。

故障一：Kernel Panic (内核恐慌)

首先出现的是 Kernel Panic，报错信息赫然写着： attempt to kill the idle task!

这是 Linux 内核在遇到无法处理的致命错误时（一般是硬件错误），为了保护数据安全而采取的一种**“紧急停机”**机制。

🛠️ 解决方法：我把内存条拔下来，用橡皮擦拭了一遍金手指，重新插回去之后就好了。所以我推测是内存接触不良的问题。这种情况先后出现了两次。

故障二：CPU 死锁

过了几天，又出现了 CPU 卡死的情况。是 soft lockup 还是 hard lockup 我记不清了，大概就是 CPU 的一个核遇到了 Bug，陷入了无限循环或者等待中。

系统的看门狗定时器（Watchdog）发出了相应警报，cpu#0 和 cpu#3 先后卡死，我真服了。

故障三：文件系统损坏
在第二次处理 Kernel Panic 的时候，又遇到了硬盘问题：fsck Error。 大概是文件系统出错了，它的索引指示和实际存储不一致。虽然用系统自带的 fsck 修复工具手动修复了一下就能进系统，但这明显是强制关机的后遗症。

以上这些问题，直接导致了我很长一段时间不接受二手的电子产品——感觉有点亏了。

☁️ 终结：云端的崩塌

就是这样不断的修复，缝缝补补又三年。

直到 2025 年 11 月 18 号 前几天，Docker 突然打不开了。因为我的网站是跑在 Docker 容器里的，这直接导致网站失联。经过检查，是因为设置的 IP 地址和现有设备冲突导致的网络问题。当时手头有其他事情，就没及时解决。

然而，压死骆驼的最后一根稻草来了。

到了 18 号，Cloudflare 居然崩溃了，导致了全球接近一半的互联网宕机 4 小时（不过中国内地的网络基本没受影响，所以大家可能感觉不到）。

不幸的是，我第一个博客用的正是 CF 的 CDN 服务，所以也跟着“原地爆炸”了。这件事可以说让我彻底放弃了维护那个老旧的博客。

🚀 重生：拥抱 Hugo 与 GitHub Pages

后来，我又看见学长使用了静态网站部署在 GitHub Pages 上的方法。

我觉得这很有极客感。我认为这种安全、稳定且免费的静态网站，才更加适合不需要数据库交互的个人用户。

最后我选择了 Hugo。

  • 它主要由 Go 语言编写，部署简单。
  • 性能强大，在内容较多时，生成速度显著快于 Hexo 等其他框架。
  • 技术门槛在静态框架中较低，非常适合尝试。

于是，这个你正在浏览的网站——**「天际私语」**诞生了。它是我技术探索的新起点，也希望它能比它的“前任”活得更久一些。

有些感受，存在于语言的边界之外； 有些共鸣，发生在沉默的瞬间。 当真正的理解达成时，一切表达都显得多余。

表达，为了无法表达的共鸣——欢迎来到「天际私语」
